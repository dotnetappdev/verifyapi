üéØ Overview

The HTTP Monitoring feature in Palis allows you to intercept and inspect HTTP traffic from your applications in real-time. Similar to HTTP Toolkit, this powerful tool enables you to view request/response headers, modify responses, and debug API communication issues.

‚ú® Key Features

  • Traffic Interception: Capture all HTTP requests passing through the proxy
  • Real-time Inspection: View headers, body, and metadata as requests happen
  • Response Modification: Modify responses before they reach your application
  • Method Color Coding: Quickly identify request types with color-coded badges
  • Configurable Endpoints: Target specific API endpoints for monitoring
  • Auto-forward Mode: Choose between manual and automatic request forwarding

üöÄ Getting Started

1Open the Monitoring Panel

Click the Monitor button in the toolbar or select the Monitoring tab in the sidebar to open the HTTP monitoring panel.

2Configure the Proxy

Before starting the proxy, configure the following settings:

  • Proxy Port: Default is 8888. Choose any available port.
  • Target Endpoints: Add the URLs where requests should be forwarded (e.g., http://localhost:3000)
  • Enable Traffic Interception: Check to capture and inspect traffic
  • Auto-forward Requests: Check to automatically forward requests without manual intervention
Example Configuration: Port: 8888 Target Endpoint: http://localhost:3000 Interception: Enabled Auto-forward: Disabled (for manual response modification)

3Start the Proxy Server

Click the ‚ñ∂ Start Proxy button. Once started, you'll see a status indicator showing the proxy is active and the port it's listening on.

üí° Tip: Configure your application to use the proxy by setting the HTTP proxy to http://localhost:8888 (or your chosen port).

4Capture and Inspect Traffic

Once your application makes requests through the proxy, they'll appear in the traffic list. Click on any request to view detailed information:

üìä Understanding the Interface

Traffic List (Left Panel)

Shows all captured requests with:

  • GET POST PUT DELETE Method badges color-coded for easy identification
  • Status Code: HTTP response status (200, 404, 500, etc.)
  • Timestamp: When the request was made
  • URL: Full request URL with query parameters
  • Metrics: Response time and size

Request Details (Right Panel)

Detailed view of the selected request:

  • Request Information: Method, URL, timestamp, client address
  • Request Headers: All HTTP headers sent by the client
  • Request Body: Formatted JSON or raw body content
  • Response Information: Status code, response time, size
  • Response Headers: All HTTP headers returned by the server
  • Response Body: Formatted JSON or raw response content

Manual Response Modification

When auto-forward is disabled, you can modify responses before they reach the client:

  1. Select an intercepted request (without a response)
  2. Click the ‚úèÔ∏è Modify Response button
  3. Edit the status code, status text, and body

üéØ Best Practices

  • Start with auto-forward enabled to understand normal traffic patterns
  • Use the üóëÔ∏è Clear button regularly to remove old traffic from the list
  • Configure specific target endpoints rather than catch-all proxying
  • Use descriptive names for target endpoints (e.g., "API Server", "Auth Service")
  • Stop the proxy when not in use to free up the port
  • Review captured traffic regularly to identify performance bottlenecks

üîç Troubleshooting

Proxy won't start

  • Check if another application is using the selected port
  • Try a different port number (e.g., 8889, 9000)
  • Restart the application

No traffic appearing

  • Verify your application is configured to use the proxy
  • Ensure "Enable Traffic Interception" is checked
  • Check that the target endpoint is correctly configured
  • Verify the proxy server is running (green status indicator)

Requests timing out

  • Enable "Auto-forward Requests" if you're not manually responding
  • Check that the target endpoint is reachable
  • Verify no firewall is blocking the proxy port

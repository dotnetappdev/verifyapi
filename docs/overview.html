üîÑ Typical Workflow

1. Authentication & Setup

Login with your credentials or create a new account. The app uses secure local authentication with encrypted password storage.

2. Create Collections

Organize your API requests by creating collections for different projects or API endpoints. Use folders for further organization.

3. Build Requests

Create new HTTP requests by specifying the method, URL, headers, and body. Configure authentication as needed.

4. Send & Analyze

Send requests and analyze responses. Review status codes, headers, response times, and JSON data formatting.

5. Write Tests

Add test scripts to validate API responses automatically. Use the Visual Studio-style assertion framework.

6. Run Test Suites

Execute individual tests or entire test suites. Review detailed test results and performance metrics.

üîí Security Features

üè† Local-First Architecture

Passwords and sensitive data are encrypted using industry-standard AES-256 encryption with secure key management.

üõ°Ô∏è Context Isolation
